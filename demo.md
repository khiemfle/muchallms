# Demo Recording Script

Goal: show Muchallms managing multiple LLM tabs, broadcasting a prompt, saving history, and reopening a past conversation with stored URLs.

## Recording setup
- Browser: Chrome (or Arc/Brave) with the extension loaded.
- Windows: empty desktop, no other popups.
- Accounts: logged in to ChatGPT, Claude, Gemini, Grok, Perplexity.
- Display: 1440p preferred, cursor visible.
- Prep: close all LLM windows; open the controller from the extension icon.

## Script (approx 90-120s)

### 0:00 - 0:10 Intro
- On screen: Chrome toolbar with the extension icon.
- Voice: "This is Muchallms, a lightweight controller for managing multiple LLM windows from one place."
- Action: click the extension icon to open the controller window.

### 0:10 - 0:30 Open LLM grid
- On screen: controller UI.
- Voice: "Open the LLM grid in one click."
- Action: click "Open LLMs".
- Show the grid of LLM windows opening.

### 0:30 - 0:55 Broadcast a prompt
- On screen: controller prompt box.
- Voice: "Type once, send to all."
- Action: paste prompt: "Draft a one-sentence product tagline for a team calendar app."
- Action: click the send icon or press Cmd/Ctrl+Enter.
- Show each LLM responding.

### 0:55 - 1:10 Conversation history
- On screen: controller header menu.
- Voice: "Muchallms keeps a local history of your broadcasts."
- Action: open History; expand the latest conversation; show per-LLM links.

### 1:10 - 1:25 Reopen stored conversation
- On screen: History list.
- Voice: "Reopen a past conversation using the stored URLs for each provider."
- Action: click "Open" on the latest conversation.
- Show LLM windows reopening to the stored conversation URLs.

### 1:25 - 1:35 Privacy note + close
- On screen: controller.
- Voice: "Everything stays in your browser. No external servers."
- Action: optional: open Privacy policy modal.

### 1:35 - 1:45 Outro
- On screen: grid of LLM windows + controller.
- Voice: "Muchallms keeps multi-LLM work fast, focused, and organized."

## Notes for the recording
- Keep the prompt short so all providers respond quickly.
- If a provider is slow, pause briefly or cut between shots.
- Avoid typing errors; paste the prompt to keep pacing tight.
